# VDM Phase 1.1 — Capability Negotiation (proven in Python) — Design Spec

**Date:** 2026-06-15
**Status:** Approved (design); pending user review → implementation plan
**Parent specs:** `Vector_Drift_Monitor_Spec_v2.md` (Draft v0.2) §5–6, §9; `2026-06-15-vdm-phase0-spike-design.md`
**Builds on:** the completed `vdm-spike/` Phase 0 repo (signal validated, GO).
**Scope:** One sub-project — the first slice of PRD Phase 1, proving the capability-negotiation moat in Python before any TypeScript port.

---

## 1. Purpose

Phase 0 proved the drift *signal* works. Phase 1.1 proves the *moat*: that a single adapter contract plus **capability negotiation** lets the engine pick the richest analysis each store can support and degrade honestly when it can't. The PRD (§12.5) prescribes exactly this sequencing — prototype in Python, prove the architecture, then port to Option A (TS control plane + Python compute). This slice does the "prove the architecture" step and explicitly **defers** the TS port.

It also resolves the open Phase 0 finding: **Finding A** (retrieval-RBO is only valid on an incrementally-evolving index) by rebuilding the scenarios to model incremental evolution.

## 2. Decisions locked during brainstorming

| Decision | Choice | Rationale |
|---|---|---|
| Language / base | Pure Python, extend `vdm-spike/` | Reuse all Phase 0 detectors; defer TS port until the contract is proven (PRD §12.5). |
| QUERY-plan store | **Fake** query-only adapter (not real Pinecone) | Reproducible, CI-friendly, zero cloud cost; the goal is proving negotiation logic, not a vendor. Pinecone can actually return vectors (PRD v0.2 correction), so a deliberate fake is a cleaner QUERY-only example. |
| Conformance depth | Core: capability honesty + sampling correctness + query contract | The essential trust mechanism without gold-plating; round-trip/resilience/badge deferred. |
| Realistic scenarios | Folded into this slice | Resolves Finding A and makes the QUERY-plan retrieval drift testable. |
| Plan coverage | All three branches (FULL, QUERY, MINIMAL) | Two real-SDK FULL adapters + two tiny fakes cover every negotiator branch. |

## 3. Success criteria

1. The negotiator selects **FULL** for pgvector & Qdrant, **QUERY** for the query-only adapter, **MINIMAL** for the no-query adapter — verified by unit tests.
2. Each plan runs **only its allowed detector families**; QUERY-plan output carries a **sampling-confidence/bias flag** (query-seeded sample never presented as an unbiased distribution).
3. The **conformance suite passes for all four adapters** AND **fails a deliberately-dishonest adapter** (a test proving the suite has teeth).
4. All **Phase 0 detectors are reused unchanged** (no detector logic edited).
5. Realistic incremental-evolution scenarios make **retrieval-RBO a properly-gated signal again** (null/benign retrieval quiet; harmful retrieval fires), closing Finding A.

## 4. Architecture

Extends the existing `vdm-spike/` package — all Python, all in-memory or local (no new external services).

```
src/vdm_spike/
  adapters/
    __init__.py
    base.py        # VectorStoreAdapter (Protocol), Capabilities, StoreDescriptor, ProbeResult, Scope, QueryHit
    pgvector.py    # PgVectorAdapter  — refactor of today's store.py            → FULL
    qdrant.py      # QdrantAdapter    — qdrant-client in-memory (no Docker)      → FULL (2nd SDK)
    fakes.py       # FakeQueryOnlyAdapter → QUERY ; FakeMinimalAdapter → MINIMAL
  negotiation.py   # DriftPlan enum, select_drift_plan(caps), PLAN_FAMILIES map
  sampling.py      # query_seeded_sample() + native/reservoir wrappers (budget-bounded)
  conformance.py   # run_conformance(adapter) -> ConformanceReport (reusable battery)
  scenarios.py     # REWRITE → incremental-evolution model
  run.py           # REWRITE → per-adapter: negotiate plan → run plan-appropriate families → flag confidence
  # unchanged: core.py, features.py, corpus.py, embed.py, ops.py, power.py, report.py,
  #            detectors/distribution.py, detectors/retrieval.py
```

`store.py` is refactored into `adapters/pgvector.py` (the first conforming adapter); `QueryHit` moves to `adapters/base.py`.

### 4.1 The adapter contract (`adapters/base.py`)

Mirrors PRD §5.1 / §5.2, Python `Protocol`. `Scope` is a namespace string for this slice (kept simple; matches Phase 0).

```python
class VectorStoreAdapter(Protocol):
    def describe(self) -> StoreDescriptor: ...
    def capabilities(self) -> Capabilities: ...
    def count(self, scope: str) -> int: ...
    def sample(self, scope: str, k: int) -> Snapshot: ...          # capability-gated
    def fetch_by_ids(self, ids: list[str], scope: str) -> Snapshot: ...
    def query(self, vector, scope: str, k: int) -> list[QueryHit]: ...
    def probe(self) -> ProbeResult: ...
```

`Capabilities` fields used this slice: `returns_vectors`, `unbiased_sample`, `live_query`, `id_listing`, `random_sample` (`'native'|'reservoir'|'query-seeded'|'none'`), `max_batch`. Declared, not inferred; verified by the conformance suite.

## 5. Negotiation & plan→families (`negotiation.py`)

PRD §6 logic with the v0.2 correction (`unbiased_sample` gates distribution drift, not merely `returns_vectors`):

```python
def select_drift_plan(c: Capabilities) -> DriftPlan:
    if c.returns_vectors and c.unbiased_sample:
        return DriftPlan.FULL      # distribution (A) + retrieval (B) + ops (C)
    if c.live_query:
        return DriftPlan.QUERY     # retrieval (B) + ops (C); retrieval via query-seeded sample, flagged biased
    return DriftPlan.MINIMAL       # ops (C) only
```

`PLAN_FAMILIES` maps each plan to the detector families it may run. `run.py` negotiates the plan per adapter, runs only the allowed families, and the verdict output labels the **active plan** and a **sampling-confidence caveat** for query-seeded sampling. Schema-drift (family D) is out of scope.

## 6. Adapters & conformance

- **PgVectorAdapter** (FULL) — refactor of Phase 0 `store.py`. `returns_vectors=True, unbiased_sample=True, live_query=True`.
- **QdrantAdapter** (FULL) — `qdrant-client` run in-memory (`QdrantClient(location=":memory:")`), no Docker. Same plan as pgvector via a different SDK → proves genericity. Same capabilities.
- **FakeQueryOnlyAdapter** (QUERY) — wraps an in-memory vector set; declares `returns_vectors=False, unbiased_sample=False, live_query=True`. `sample()` raises `CapabilityError` (gated); `query()` works → forces query-seeded sampling.
- **FakeMinimalAdapter** (MINIMAL) — `live_query=False` too → ops only.
- **Conformance suite** (`run_conformance(adapter) -> ConformanceReport`): for any adapter verifies
  1. **Capability honesty** — `returns_vectors=True` ⟹ `sample()` returns declared-dim vectors; `False` ⟹ it does not (raises/empty).
  2. **Sampling correctness** — `sample(k)` returns ≤k records, de-duplicated, all within `count()`.
  3. **Query contract** — `query()` returns ranked hits with monotonic ranks and scores in the metric's expected range.
  A dedicated test points the suite at a deliberately-dishonest adapter (declares `returns_vectors=True` but returns nothing) and asserts the report **fails** — proving the suite has teeth.

## 7. Query-seeded sampling (`sampling.py`)

For QUERY-plan adapters that can't return an unbiased sample, `query_seeded_sample()` issues the scenario's seed-query panel, collects returned neighbor IDs + scores as a **pseudo-sample**, and tags it `biased=True`. It powers retrieval + score-distribution drift only — never distribution stats. Native/reservoir wrappers cover the FULL adapters. All strategies honor a sampling budget (max records).

## 8. Realistic scenarios (`scenarios.py` rewrite — resolves Findings A & B)

Model an **incrementally evolving index**:

| Scenario | Construction | Gated expectation |
|---|---|---|
| null-control | `current` = **identical** corpus (same docs, same vectors) | all families quiet (incl. retrieval) |
| benign-growth | baseline docs **retained** + ~10% same-distribution additions | distribution + retrieval quiet |
| topic-shift | retained docs + injected off-topic docs that displace top-k | mmd + classifier + retrieval fire |
| model-swap | retained docs re-embedded with a **norm-divergent** model pair | mmd + classifier + retrieval fire; **norm_ks gated if the pair gives verified clean norm divergence**, else corroborating |
| broken-writes | retained docs with zero-norm vectors injected | ops fires |

The norm-divergent model pair is chosen and **verified by a test** in the plan; if no clean local pair is available, norm_ks stays corroborating (informational) and that is documented — same honesty discipline as Phase 0.

## 9. Testing approach

TDD throughout:
- **negotiation:** unit test each branch (FULL/QUERY/MINIMAL) from synthetic `Capabilities`.
- **conformance:** parametrized across all four adapters (pass) + the dishonest adapter (fail) → proves teeth.
- **adapters:** pgvector against live local Postgres (existing fixture); Qdrant + fakes fully in-memory.
- **query-seeded sampling:** asserts the pseudo-sample is tagged biased and powers retrieval/score drift only.
- **gate:** re-run across FULL adapters with the realistic scenarios; harmful fire, benign/null (incl. retrieval) quiet.
- All Phase 0 tests continue to pass unchanged.

## 10. Out of scope (YAGNI)

No TypeScript / control-plane port; no baseline persistence or scheduler; no API server; no UI; no alerting; no real Pinecone/Vertex; no schema-drift family (D); no new operational metrics (cardinality/latency) beyond Phase 0; `Scope` stays a namespace string (no metadata-filter scopes yet).

## 11. Deliverable

An extended `vdm-spike/` where a single command negotiates a drift plan per adapter, runs the plan-appropriate detector families with honest confidence flags, passes a conformance suite proven to have teeth, and demonstrates retrieval-RBO restored as a gated signal under realistic incremental-evolution scenarios — the evidence that the adapter-contract + capability-negotiation architecture is sound enough to port to Option A.
